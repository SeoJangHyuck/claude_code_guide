## 슬라이드 6-3 — AFTER ② rule (1분 30초)

**핵심 메시지**: "경로 기반 규칙 — reports/ 폴더 작업할 때만 자동 발동."

**본문 콘텐츠**:

```
② rule

  .claude/rules/reports.md
  paths: "reports/**"
  ─────────────────────────────
  슬라이드 수: 최대 5장
  수치: 전년 대비 변화율 함께 표기

  → reports/ 폴더 밖 → 규칙 비발동
  → reports/ 폴더 지정 → 규칙 자동 발동

   📺 시연 화면 — VSCode + Claude Code 확장
```

**발표 노트**:
> "다음은 rule입니다. .claude/rules 폴더에 reports.md라는 파일이 있어요. 맨 위에 paths 한 줄 — 'reports 폴더 안에서 작업할 때만'이라고 적혀 있습니다. 슬라이드 수 제한, 변화율 표기, 결론 슬라이드 필수 같은 보고서 전용 규칙이에요. (경로 바꿔서 확인) reports 폴더 밖에서 작업하면 이 규칙이 켜지지 않습니다. reports 폴더를 지정하면 그 순간 자동으로 발동됩니다. CLAUDE.md가 항상 켜져 있는 규칙이라면, rule은 '특정 자리에서만' 켜지는 규칙입니다."

**시각 표현 메모**:
- ② 번호 강조. paths 한 줄 강조색.
- "밖 → 비발동 / 안 → 자동발동" 대비 표현.

## 슬라이드 6-5 — AFTER ③ skill 결과 확인 (2분)

**핵심 메시지**: "출력: Cover + 본문 3장 + Closing = 5장. DESIGN_GUIDE 자동 준수 + 자가 검증 통과."

**본문 콘텐츠**:

```
PPT 결과 확인 — reports/부산_인구현황.pptx

  슬라이드 1  Cover           다크 네이비 배경 + BDI 로고 + 큰 타이틀
  슬라이드 2  주민등록인구 비교  풀폭 막대차트 (헤더 스트립 X)
  슬라이드 3  순이동인구       좌·우 데이터 표 (카드·이모지 X)
  슬라이드 4  합계출산율+시사점  좌 표 / 우 들여쓰기 리스트 (번호 grid X)
  슬라이드 5  Closing         다크 + 큰 결론 + 인라인 통계 (KPI grid X)

  ─ DESIGN_GUIDE.md 자동 적용 ─
    · 풀폭 컬러 스트립 X · 액센트 바 X · 카드 좌측 보더 X
    · 그림자·라운드 코너 X · 이모지 X · KPI grid X
    · 강조색은 데이터에만 (변화율 +/-)
    · 폰트: Pretendard 단일

  → verify_design.py 자동 실행 → 127 shape / 0 결함 ✅

   📺 시연 화면 — 생성된 PPT 파일 열기 + 검증 결과
```

**발표 노트**:
> "(PPT 파일 열기) 슬라이드가 5장 나왔습니다. 보시면 — 흔히 보는 AI가 만든 PPT 같지 않죠? 풀폭 컬러 띠도 없고, 카드 좌측에 강조 보더도 없고, 큰 숫자 박스 3개로 KPI 자랑하지도 않습니다. 이게 우연이 아니라 — skill 폴더 안에 DESIGN_GUIDE.md라는 디자인 룰 문서가 있어서, 그 안에 11종 AI 안티패턴이 명시돼 있습니다. 빌드할 때마다 이 룰을 따라 만들고, skill 절차 안에서 verify_design.py가 9개 검사 항목으로 자동 점검합니다. 방금 127개 shape를 검사해서 결함 0건. 이게 통과돼야 파일이 저장됩니다. 저장 단계 자동 검증이 어떻게 작동하는지는 잠시 후 hook 설명에서 자세히 보겠습니다. 결과적으로 *기계가 만들었지만 사람이 만든 듯한* 보고서가 나옵니다."

**시각 표현 메모**:
- 5장 구성을 한 줄씩 표 형태로 (Cover / 본문 3 / Closing).
- 하단에 "DESIGN_GUIDE.md 자동 적용" 박스 — 제거된 안티패턴 리스트.
- "127 shape / 0 결함" 검증 결과 강조.
- 단일 강조색(BDI 네이비)·Pretendard 폰트 표기.

## 슬라이드 6-7 — AFTER ④ hook — UserPromptSubmit (2분)

**핵심 메시지**: "질문하는 그 순간 — 데이터 컨텍스트가 보이지 않게 끼어 들어감."

**본문 콘텐츠**:

```
④ hook — UserPromptSubmit

  .claude/settings.json
  ─────────────────────────────
  "UserPromptSubmit" → inject_context.sh 자동 실행

  내가 채팅에 한 줄 입력하는 순간:

    [자동 컨텍스트 — busan_data]
    📅 데이터 갱신일: 2026-05-19
    📂 파일 17개 (최신: 부산_인구통계_기장군.xlsx)
    ✅ (출처: 파일명) 형식으로 자동 표기 가능

  → 사용자 화면엔 안 보임, 모델에게만 전달
  → "출처 써줘" 지시 없이도 출처가 자동으로 붙는 진짜 이유

   📺 시연 화면 — VSCode + Claude Code 확장
```

**발표 노트**:
> "두 번째 hook은 UserPromptSubmit입니다. 제가 채팅창에 메시지를 입력하고 엔터를 누르는 순간 — 모델한테 메시지가 전달되기 직전에 — inject_context.sh가 자동으로 실행돼서, busan_data 폴더의 갱신일과 최신 파일명이 컨텍스트로 끼어 들어갑니다. 6-2에서 '출처 써줘'라고 안 했는데도 응답에 출처가 자동으로 붙었던 거 기억하시죠? 그건 CLAUDE.md 규칙 *플러스* 이 hook이 함께 작동한 결과입니다. CLAUDE.md는 '출처를 써라'는 규칙이고, 이 hook은 '쓸 출처가 어디 있는지'를 매번 자동으로 알려줍니다. 사용자 화면엔 이 컨텍스트가 안 보여요 — 모델한테만 전달됩니다."

**시각 표현 메모**:
- ④ 번호 강조. UserPromptSubmit 굵게.
- "내가 채팅에 한 줄 입력하는 순간" 강조.
- 자동 컨텍스트 블록은 배경색 다르게 (모델에게만 보이는 메시지 느낌).
- 6-2와 연결되는 화살표 또는 콜아웃 추천.

## 슬라이드 6-8 — AFTER ④ hook — PostToolUse (2분)

**핵심 메시지**: "저장 직전 — 출처 누락·이상값 자동 차단."

**본문 콘텐츠**:

```
④ hook — PostToolUse

  .claude/settings.json
  ─────────────────────────────
  Write 도구로 reports/*.pptx 저장 직전
  → verify_report.sh 자동 실행

  출처 누락 슬라이드 감지 → 저장 차단
  검증 통과 → 저장 허용

   📺 시연 화면 — 차단 + 통과 장면
```

**발표 노트**:
> "세 번째 hook은 PostToolUse입니다. PPT 파일이 reports 폴더에 저장되기 직전에 verify_report.sh가 자동으로 실행돼요. 일부러 출처 없는 수치를 한 슬라이드에 끼워 넣어 보겠습니다. (시연) 보시죠 — 저장이 차단됩니다. '2번 슬라이드 출처 누락'이라고 알려줘요. 수정하고 다시 저장하면 — 검증 통과, 파일이 저장됩니다. 정리하면 — SessionStart는 *시작 시점*, UserPromptSubmit는 *질문 시점*, PostToolUse는 *저장 시점*. hook 하나면 한 순간만 잡지만, 세 개를 같이 쓰면 작업의 앞·중·뒤가 전부 자동화됩니다."

**시각 표현 메모**:
- ④ 번호 강조. "차단 → 수정 → 통과" 3단계 흐름 표현.
- SessionStart / UserPromptSubmit / PostToolUse 3시점 ("시작 / 질문 / 저장") 한 줄 요약 표시.

## 슬라이드 6-9 — Before / After (1분 30초)

**본문 콘텐츠**:
