> 톤 가이드: 두 단계 객관 비교 + 담백 전환. (1) 5개 도구를 4축으로 펴놓되, 표 위에 도구별 *특화 영역* 한 줄로 각자 잘하는 게 다르다는 점부터 인정 (4-1). (2) 그 위에서 갈리는 결정적 차이 4가지를 *워크플로우 순서*(진입→작업→자산화·검증→공유)로 카드화 (4-2). (3) 비유 없이 명제형 짧은 메시지로 다음 챕터 연결 (4-3).

> 모든 데이터: 2026-05 기준 공식 문서 (anthropic.com, openai.com, deepmind.google, cursor.com). 발표 직전 한 번 더 검증.

## 슬라이드 4-2 — 결정적 차이 4가지 (3분 30초)

**핵심 메시지**: "1번 축은 다 강하다. 2~4번에서 갈린다. 그 차이가 일하는 흐름 안에서 어떻게 박히는지 — 시작·작업·자산화·공유 순서로 4가지."

**본문 콘텐츠**:

```
─────────────────────────────────────────────────────────────────────
[시작 / 진입]
① 터미널 없이 VSCode 확장 한 줄로 시작

   • 검은 터미널 X — 파일 트리·메뉴·마우스 그대로
   • Claude Code 연동 한 번에 (확장 설치 = 연동 완료)
   • VSCode 마켓 확장도 함께 씀
     예: Markdown All in One · vscode-pdf · Code Spell Checker
       (각 5M+ 다운로드)
   ─────────────────────────────────────────
   → 5장 진입 가이드 + 6장 시연도 VSCode 확장으로
─────────────────────────────────────────────────────────────────────
[작업 / 보안]
② 작업을 클라우드 연동 없이 로컬에서

   • 파일이 내 PC에서 그대로 처리 — 클라우드 업로드 자체가 없음
   • 채팅형 도구(ChatGPT·Gemini·Claude 웹)는 학습 옵트아웃은 되지만,
     작업 시작 시 파일은 클라우드로 올라감
   ─────────────────────────────────────────
   → 5장 settings.json에서 권한·자동 가드 설정
─────────────────────────────────────────────────────────────────────
[자산화 / 검증]
③ 같은 작업을 매주 다시 짜지 않고, 환각이 박히기 전에 막는다

   • Skills — 한 번 짜둔 절차를 매주 호출
     (반복 분석·자료 정리·요약 작업)
   • Subagents — 긴 작업·여러 자료 병렬 처리 (격리된 컨텍스트)
   • Hook 29개 이벤트 — 파일 저장 직전·도구 호출 전 자동 검증
     (환각이 파일에 박히기 전 차단, settings.json 한 줄로 동작)
   ─────────────────────────────────────────
   → 5장 skills · agents · hooks
─────────────────────────────────────────────────────────────────────
[공유]
④ 동료가 만든 절차를 git 또는 plugin으로 받아쓴다

   • git — `.claude/` 폴더 통째로 공유
     (그 안에 절차·권한·외부 연결까지 다 들어감)
   • Plugin — 패키지로 묶어 marketplace 배포, 한 줄로 설치
   • MCP 오픈 표준 — 다른 도구가 만든 연결도 그대로 흡수
   • GitHub OSS 자산도 동일 흐름
     (awesome-claude-code · claude-code-skills · plugins)
   ─────────────────────────────────────────
   → 7장 사내 공유 flow (파일 → zip → git → marketplace 4단계)
─────────────────────────────────────────────────────────────────────
```

**발표 노트 (3분 30초)**:
> "1번 축은 다 강하다고 말씀드렸죠. 그러면 2~4번 축의 차이가 일하는 흐름 안에서 *어떻게 박히는지* — 4가지로 정리했습니다. 순서는 일하는 흐름을 따라갑니다. 시작, 작업, 자산화, 공유.
>
> **첫 번째, 시작.** Claude Code는 VSCode 확장 한 줄로 들어옵니다. 검은 터미널 안 나옵니다. 파일 트리·메뉴·마우스 그대로예요. 확장 설치하면 연동 끝나고, 곁들여서 VSCode 마켓에 있는 다른 확장들 — Markdown All in One, vscode-pdf, Code Spell Checker 같은 거 — 함께 씁니다. 다 다운로드 500만 이상인 인기 확장들이에요. 6장 시연도 VSCode 확장 안에서 합니다.
>
> **두 번째, 작업.** 보안 얘기예요. 방금 비교표에서 보셨죠. Claude Code는 작업이 로컬에서 돌아갑니다. 파일이 내 PC에서 그대로 처리돼요 — 클라우드 업로드 자체가 없습니다. 비교를 위해 짚으면, 채팅형 도구(ChatGPT·Gemini·Claude 웹)는 학습 옵트아웃을 켜놓아도 *작업을 시작하려면 파일을 일단 클라우드에 올려야* 합니다. 옵트아웃은 "올린 데이터로 학습 안 함"이지 "올리지 않음"이 아니에요. Claude Code는 그 단계 자체를 건너뜁니다.
>
> **세 번째, 자산화 + 검증.** 같은 작업을 매주 다시 짜지 않게 만드는 도구가 셋입니다. Skills로 절차를 한 번 짜두면 매주 호출됩니다. Subagents는 긴 작업·여러 자료를 격리된 컨텍스트에서 병렬로 돕습니다. 그리고 Hook — 29개 이벤트에 자동 검증을 걸 수 있어요. 파일 저장 직전, 도구 호출 전, 세션 종료 시 같은 시점에 자동으로 도는 검증입니다. 환각이 파일에 박히기 전에 막아주는 장치예요. settings.json에 한 줄 적기만 하면 동작합니다.
>
> **네 번째, 공유.** 동료가 만든 절차를 받아쓰는 길이 두 가지입니다. 하나는 git — `.claude` 폴더를 통째로 공유하면 그 안에 절차·권한·외부 연결까지 다 들어 있어요. 또 하나는 Plugin — 패키지로 묶어서 marketplace에 올리고 한 줄로 설치합니다. MCP가 오픈 표준이라 다른 도구가 만든 연결도 그대로 흡수되고요. GitHub에 OSS 자산도 빠르게 쌓이고 있습니다 — awesome-claude-code, claude-code-skills, plugins 같은 리포들이에요.
>
> 이 4가지가 시작부터 공유까지 흐름 안에서 *결정적으로* 차이가 갈리는 지점입니다. 다음 5장에서 이 흐름이 *실제로 들어가는* `.claude` 폴더를 같이 열어보겠습니다."

**시각 표현 메모**:
- 카드 4개 2×2 그리드 (기존 `.diff-grid.cards-4` 재사용).
- 각 카드 상단에 *워크플로우 단계 라벨* — `[시작]` `[작업]` `[자산화·검증]` `[공유]` 알약형 태그 (새 CSS `.step-label`).
- 카드별 강조 색 차등 (4색) — 단계 색으로 흐름 시각화.
- 카드 하단 "다음 챕터 연결" 화살표 (기존 `.diff-link` 재사용).
- 그림 없음.
